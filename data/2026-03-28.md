### [1] [SHADOW: Seamless Handoff And Zero-Downtime Orchestrated Workload Migration for Stateful Microservices](https://arxiv.org/abs/2603.25484)
*Hai Dinh-Tuan*

Main category: cs.DC

TL;DR: SHADOW is a Kubernetes-native framework that enables zero-downtime migration of stateful microservices by using shadow pods and message replay, eliminating service downtime and reducing migration time by up to 77% compared to sequential approaches.

  <summary>Details</summary>
Motivation: Current StatefulSet migration in Kubernetes causes significant service downtime (median 38.5s) due to sequential stop-recreate cycles and identity constraints that prevent concurrent pod operation, requiring a solution for seamless stateful workload migration.

Method: SHADOW implements the MS2M approach as a Kubernetes Operator using ShadowPod strategy: creates a shadow pod from CRIU checkpoint on target node while source pod continues serving, with concurrent message replay. For StatefulSets, uses identity swap with ExchangeFence mechanism to re-checkpoint shadow pod, create StatefulSet-owned replacement, and drain message queues for zero message loss.

Result: Evaluation on bare-metal Kubernetes with 280 migration runs across 4 configurations and 7 message rates (10-120msg/s) shows: ShadowPod reduces restore phase by up to 92%, eliminates service downtime entirely, reduces total migration time by up to 77% compared to sequential baseline, with zero message loss across all 280 runs.

Conclusion: SHADOW successfully enables zero-downtime migration of stateful microservices in Kubernetes through shadow pods and message-based state transfer, solving the state management and identity constraint problems of StatefulSet workloads while guaranteeing zero message loss.

Abstract: Migrating stateful microservices in Kubernetes requires careful state management because in-memory state is lost when a container restarts. For StatefulSet-managed workloads, the problem is compounded by identity constraints that prohibit two pods with the same ordinal from running simultaneously, forcing a sequential stop-recreate cycle with a median 38.5s of service downtime. This paper presents SHADOW Seamless Handoff And Zero-Downtime Orchestrated Workload Migration, a Kubernetes-native framework that implements the Message-based Stateful Microservice Migration (MS2M) approach as a Kubernetes Operator. SHADOW introduces the ShadowPod strategy, where a shadow pod is created from a CRIU checkpoint image on the target node while the source pod continues serving traffic, allowing concurrent operation during message replay. For StatefulSet workloads, an identity swap procedure with the ExchangeFence mechanism re-checkpoints the shadow pod, creates a StatefulSet-owned replacement, and drains both message queues to guarantee zero message loss during the handoff. An evaluation on a bare-metal Kubernetes cluster with 280 migration runs across four configurations and seven message rates (10--120msg/s) shows that, compared to the sequential baseline on the same StatefulSet workload, the ShadowPod strategy reduces the restore phase by up to 92%, eliminates service downtime entirely, and reduces total migration time by up to 77%, with zero message loss across all 280 runs.
